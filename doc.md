我要做一个整蛊人用的小程序（最好是可以在手机上用的）。大致内容如下：

让用户传自己的脸照，然后假装使用AI分析，然后输出一个全身体检报告。

这个体检报告要足够详细，分为几个大板块，然后重点就是，脑功能模块中的认知水平、智力水平要是轻度障碍。

先规划一下这个idea怎么实现吧，更新在这个文档下面

---

## 实现规划

### 1. 产品定位

| 维度 | 选择 |
|------|------|
| 形态 | **移动端优先的 H5 网页**（PWA 可选），浏览器打开即用，方便微信/QQ 发链接整蛊 |
| 是否需要真 AI | **不需要**。上传照片后走「假分析」流程，报告由模板 + 随机数生成 |
| 核心笑点 | 报告整体看起来很专业、指标大多正常，唯独 **脑功能模块 → 认知水平 / 智力水平 = 轻度障碍** |
| 合规提示 | 首页或报告页底部加一行小字：「本报告仅供娱乐，非医疗诊断」 |

**为什么不先做微信小程序：** 需要企业主体 + 审核，迭代慢；H5 零门槛分享，整蛊场景更合适。若后续要投微信生态，可把同一套前端迁到 Taro / uni-app。

---

### 2. 用户流程

```
打开链接 → 上传/拍照 → 假 AI 分析动画（15~30s）→ 全身体检报告 → 截图分享 / 保存长图
```

1. **首页**：简短说明 +「开始体检」+ 选图（`<input capture="environment">` 调起手机相机）
2. **分析页**：进度条 + 分步骤文案（见下文），期间可做本地人脸检测增强「可信度」
3. **报告页**：分板块展示，脑功能模块视觉上可略突出（或用户滚动到该节时才有「啊？」的效果）
4. **分享**：html2canvas 生成长图，或 Web Share API

---

### 3. 技术栈建议

| 层级 | 方案 |
|------|------|
| 框架 | **Vite + React** 或 **Vue 3**（单页，部署简单） |
| 样式 | Tailwind CSS，医疗报告风：白底、细线表格、小字号指标 |
| 人脸（可选） | **MediaPipe Face Landmarker** 或 **face-api.js**（纯前端，仅用于「已识别面部特征」展示，不上传服务器） |
| 报告生成 | 本地 JSON 模板 + `faker` / 自写随机区间函数 |
| 部署 | Vercel / Cloudflare Pages / GitHub Pages（静态托管，无后端） |
| 长图 | `html2canvas` 或 `dom-to-image-more` |

**隐私：** 照片全程浏览器内处理，不上传服务器——既省事又避免隐私纠纷，还能写在页面上当卖点。

---

### 4. 假分析动画（ credibility 关键）

按顺序展示 6~8 步，每步 2~4 秒，带百分比进度：

1. 正在解析面部几何特征…
2. 估算骨龄与代谢年龄…
3. 关联 cardiovascular 风险模型…
4. 扫描 micro-expression 压力指数…
5. **运行 neuro-cognitive 关联推断…** ← 可在此步 UI 多闪一下
6. 生成多系统综合报告…

可选细节：

- 上传图上加半透明网格 / 扫描线动画
- 若人脸检测成功，显示「已定位 468 个特征点」之类具体数字
- 背景播放轻微「仪器嘀嘀」音效（默认静音，用户点开）

### 5. 报告结构（板块与字段）

整体模仿真实体检 PDF：**封面摘要 + 分系统明细 + 综合结论**。

#### 5.1 报告头

- 受检者编号（随机）、检测时间、设备型号（虚构，如 `HealthScan Pro v3.2`）
- 上传照片缩略图（圆角头像框，显「已采样」）

#### 5.2 各系统模块（示例）

| 板块 | 主要指标 | 数值策略 |
|------|----------|----------|
| 一般检查 | BMI 推算、估算年龄、血压（收缩/舒张）、静息心率 | 随机但在正常或「边缘正常」区间 |
| 心血管 | 心率变异性、动脉硬化指数、LDL 估算 | 多数「正常」 |
| 呼吸 | 肺活量估算、血氧饱和度 | 正常偏高，显得健康 |
| 消化 |  gut microbiome 多样性指数（胡编） | 中等偏上 |
| 骨骼肌肉 | 骨密度 T 值、肌肉量占比 | 正常 |
| 免疫 | 炎症因子综合分、淋巴细胞比例 | 正常 |
| 内分泌 | 皮质醇节律、甲状腺相关（TSH 估算） | 正常 |
| **脑功能 / 神经认知** | **认知水平、智力水平、反应时、工作记忆、执行功能** | **认知水平、智力水平固定为「轻度障碍」**；其余可略低于平均或标黄 |
| 心理社会 | 压力指数、焦虑倾向、睡眠 quality | 可故意写「偏高」与脑功能呼应，增加喜剧感 |
| 综合评估 | 健康总分、建议复查项 | 总分 70~85；建议复查里带上「神经认知专项」 |

#### 5.3 笑点字段写法（脑功能模块）

```
认知水平（Cognitive Level）    ████░░░░░░  42/100  【轻度障碍】
智力水平（Intellectual Level）  ███░░░░░░░  38/100  【轻度障碍】
```

配套「专业」解释文案（模板二选一随机）：

- 「基于面部不对称度与微表情延迟的关联模型，推断前额叶-顶叶通路效率低于同龄参考值。」
- 「视觉-空间整合与工作记忆分项未达年龄常模下限，建议 3 个月后复测。」

其他模块用绿色「正常」、黄色「关注」即可，形成对比。

### 6. 数据与随机化规则

```ts
// 伪代码思路
interface Report {
  meta: { id, timestamp, device };
  modules: Module[];
}

function generateReport(seed?: string): Report {
  // 可选：用照片 hash 做 seed，同一张照片结果一致（更像「真检测」）
  // 各模块指标 randomInRange(normalRange)
  // 强制覆盖：
  brain.cognitiveLevel = { score: 38-45, label: '轻度障碍' };
  brain.intellectualLevel = { score: 35-42, label: '轻度障碍' };
}
```

- **同图同结果：** 对图片做简单 hash 再 seed 随机数，避免同一个人连测两次数值跳太大
- **不同人不同：** 其他指标随 seed 变化，显得每人报告不同

### 8. UI / 视觉方向

- **色调：** 白 + 浅蓝灰（医院报告感），强调色用医疗绿 / 警示琥珀
- **字体：** 中文思源黑体 / 系统 sans；数字用等宽感
- **组件：** 指标行（名称 | 数值 | 参考范围 | 状态标签）；脑功能块可用浅黄底 `#FFF8E1` 标「需关注」
- **动效：** 分析页 skeleton；报告页 stagger 淡入各板块

### 9. 开发阶段（建议顺序）

| 阶段 | 内容 | 预估 |
|------|------|------|
| **M1** | 静态报告页 + 硬编码一份样例报告，确认版式和笑点节奏 | 0.5~1 天 |
| **M2** | 报告生成器（模板 + 随机 + 强制脑功能字段） | 0.5 天 |
| **M3** | 上传 + 假分析动画 + 路由串联 | 1 天 |
| **M4** | 移动端适配、长图导出、分享文案 | 0.5 天 |
| **M5**（可选） | 人脸检测、音效、PWA、自定义域名 | 1 天 |

**MVP 交付标准：** 手机浏览器能拍照 → 看动画 → 出完整报告 → 能截图发群。

---

### 10. 整蛊体验优化

1. **入口文案：** 「AI 全身体检」「3 分钟出报告」——不要提前剧透
2. **报告前摘要：** 先展示「总体健康良好」，用户放松后再滑到脑功能
3. **分享标题：** 生成带编号链接，对方打开是同一套 UI，不会看到「整蛊」字样
4. **二次传播：** 报告底部小按钮「帮朋友测一下」跳回首页

---

### 11. 风险与边界

- 明确娱乐声明，避免被当真产生焦虑或纠纷
- 不要用真实医院 / 仪器品牌名，用虚构型号
- 照片不上传服务器，页面上写清楚
- 若公开部署，注意各国对「伪造医疗报告」的敏感表述——用「健康趣味评估」替代「体检报告」也可降低风险
